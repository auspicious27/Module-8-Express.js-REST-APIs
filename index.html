Module 7 mein humne Node.js core HTTP module se server banana seekha. Wahan routing, response headers, JSON conversion, and status codes sab manually likhna padta tha. Real projects mein agar har route ke liye itna manual code likhenge toh code long, repetitive, and confusing ho jayega.

Express.js ek ready kitchen setup hai jahan stove, plates, shelves, and workflow already organized hain. Developer ko bas route banana hai, request handle karni hai, and response bhejna hai.

4. Express Framework

Express.js Node.js ke upar built framework hai. Framework ka matlab ready structure and helper tools. Express routing easy karta hai, middleware support deta hai, JSON parsing simple karta hai, error handling organized banata hai, and APIs quickly build karne mein help karta hai.

Node.js core HTTP module mein manually request URL check karna padta hai. Express mein hum direct app.get(), app.post(), app.put(), and app.delete() use karte hain.

Class Mein Aise Explain Karo

“Node.js raw engine hai. Express.js us engine par bana hua easy driving system hai. Aapko har internal wire manually connect nahi karni padti. Express route, request, response, middleware sabko simple syntax mein handle kar deta hai.”

5. Routing and Middleware

Routing ka matlab URL ke according kaunsa code run hoga. Example: /api/courses course list dega, /api/courses/1 single course dega. Middleware ka matlab request aur final response ke beech ka checkpoint.

Real-world story: College gate par security guard middleware hai. Student andar jaane se pehle ID check hoti hai. Agar ID valid hai toh class tak jaane dete hain. Backend mein middleware logging, authentication, validation, JSON parsing, and error handling ke liye use hota hai.

Step 3 - controllers/courseController.js Banao

Controller file API ka main logic handle karti hai. Route sirf request ko controller function tak bhejega. Controller decide karega data kaise read/create/update/delete karna hai.

controllers/courseController.js 
const courses = require("../data/courses");
const validateCourse = require("../validators/courseValidator");

function getAllCourses(request, response) {
  response.status(200).json({
    success: true,
    count: courses.length,
    data: courses
  });
}

function getCourseById(request, response) {
  const courseId = Number(request.params.id);
  const course = courses.find(function (item) {
    return item.id === courseId;
  });

  if (!course) {
    return response.status(404).json({
      success: false,
      message: "Course not found"
    });
  }

  response.status(200).json({
    success: true,
    data: course
  });
}

function createCourse(request, response) {
  const validationError = validateCourse(request.body);

  if (validationError) {
    return response.status(400).json({
      success: false,
      message: validationError
    });
  }

  const newCourse = {
    id: courses.length + 1,
    title: request.body.title.trim(),
    duration: request.body.duration.trim(),
    level: request.body.level.trim()
  };

  courses.push(newCourse);

  response.status(201).json({
    success: true,
    message: "Course created successfully",
    data: newCourse
  });
}

function updateCourse(request, response) {
  const courseId = Number(request.params.id);
  const course = courses.find(function (item) {
    return item.id === courseId;
  });

  if (!course) {
    return response.status(404).json({
      success: false,
      message: "Course not found"
    });
  }

  const validationError = validateCourse(request.body);

  if (validationError) {
    return response.status(400).json({
      success: false,
      message: validationError
    });
  }

  course.title = request.body.title.trim();
  course.duration = request.body.duration.trim();
  course.level = request.body.level.trim();

  response.status(200).json({
    success: true,
    message: "Course updated successfully",
    data: course
  });
}

function deleteCourse(request, response) {
  const courseId = Number(request.params.id);
  const courseIndex = courses.findIndex(function (item) {
    return item.id === courseId;
  });

  if (courseIndex === -1) {
    return response.status(404).json({
      success: false,
      message: "Course not found"
    });
  }

  const deletedCourse = courses.splice(courseIndex, 1);

  response.status(200).json({
    success: true,
    message: "Course deleted successfully",
    data: deletedCourse[0]
  });
}

module.exports = {
  getAllCourses,
  getCourseById,
  createCourse,
  updateCourse,
  deleteCourse
};

Common Beginner Mistakes

  • app.use(express.json()) miss karna. Isse request.body undefined ho sakta hai.
  • next() custom middleware mein miss karna. Request route tak nahi jayegi.
  • POST request browser address bar se test karna. Browser address bar mostly GET request bhejta hai.
  • Route order galat rakhna. 404 middleware hamesha valid routes ke baad hona chahiye.
  • ID string ko number mein convert na karna. "1" and 1 strict comparison mein same nahi hote.
  • Validation ke bina data save karna. Dirty data API quality kharab karta hai.

Final Revision Summary
