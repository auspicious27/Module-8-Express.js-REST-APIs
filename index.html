1. Express Framework

Express.js kya hai, need kyu padi, Node.js mein iska use kyu hota hai.

1.1 Introduction

Ye Kya Hai?

Express.js Node.js ka web framework hai. Framework ka simple meaning hai ready-made structure and tools. Node.js JavaScript ko server side par run karta hai. Express.js us Node.js ke upar ek easy layer provide karta hai jisse backend routes, APIs, middleware, request body, and responses manage karna simple ho jata hai.

Iski Need Kyu Padi?

Node.js ke HTTP module se server banana possible hai, but code long ho jata hai. Example: URL check karna, status code set karna, content type set karna, JSON convert karna, wrong route handle karna, sab manually likhna padta hai. Express ye repetitive kaam easy syntax mein kar deta hai.

Ye Kis Problem Ko Solve Karta Hai?

  • Manual routing ka problem solve karta hai.
  • Request body ko easily read karne mein help karta hai.
  • Middleware system provide karta hai.
  • Error handling ko organized banata hai.
  • Large backend project ko scalable banata hai.

Node.js Mein Express Ki Zarurat Kyu Padti Hai?

Node.js engine hai. Express driving system hai. Engine powerful hota hai, but driving system ke bina car chalana difficult hoga. Waise hi Node.js powerful hai, but Express ke bina real APIs banana lengthy ho sakta hai.

Real World Mein Kahan Use Hota Hai?

Express.js login systems, dashboards, admin panels, e-commerce APIs, student portals, payment APIs, booking systems, chat apps, and mobile app backends mein use hota hai.

Companies Kahan Use Karti Hain?

Companies Express ya Express-like backend framework ka use APIs banane ke liye karti hain. Jab frontend React app ko data chahiye hota hai, woh Express backend API ko request bhej sakta hai. Mobile apps bhi Express API se data le sakti hain.

Real-Life Example

Restaurant mein customer order deta hai. Waiter order receive karta hai. Kitchen food prepare karta hai. Waiter food customer ko return karta hai. Web app mein browser request deta hai. Express route request receive karta hai. Controller logic process karta hai. Response browser ko return hota hai.

1.2 Theory - Express Background Mein Kaise Work Karta Hai?

Jab browser ya API testing tool request bhejta hai, request pehle Express app tak aati hai. Express app middleware run karta hai. Middleware request ko check/modify kar sakta hai. Uske baad Express route match karta hai. Route controller function call karta hai. Controller response create karta hai. Express response client ko bhej deta hai.

2. Routing

Routing ka matlab hai URL ke according backend ka correct function run karna.

2.1 Introduction

Routing web application ka traffic system hai. Browser server ko URL bhejta hai. Express check karta hai ki yeh URL kis route se match ho raha hai. Match milne par us route ka function run hota hai.

Need Kyu Padi?

Ek backend mein multiple pages ya APIs hoti hain. Example: home, login, students, courses, products. Har URL par same response nahi dena. Routing hume URL wise different response dene mein help karti hai.

Real-Life Example

Hospital mein reception, billing, pharmacy, doctor room alag-alag counters hote hain. Patient jis counter par jata hai, uske according kaam hota hai. Express routing bhi URL ke according correct handler tak request bhejti hai.

2.2 Theory Flow

Browser: GET /about ↓ Express App ↓ Route Table Check ↓ Matched Route: app.get("/about") ↓ Route Function Runs ↓ Response Sent

3. Middleware

Request route tak pahunchne se pehle jo function run hota hai.

3.1 Introduction

Middleware Express ka bahut important concept hai. Middleware ek function hota hai jo request aur response ke beech mein run hota hai. Ye request ko check kar sakta hai, log kar sakta hai, body parse kar sakta hai, authentication kar sakta hai, ya error handle kar sakta hai.

Need Kyu Padi?

Har route mein same code repeat karna bad practice hai. Example: har route mein request log karna hai, ya JSON body parse karni hai, ya user login check karna hai. Middleware common work ko ek jagah handle karta hai.

3.2 Theory Flow

Request ↓ Middleware 1: JSON body parse ↓ Middleware 2: Request log ↓ Route Handler ↓ Response

Middleware Function Format

Syntax 
function middlewareName(request, response, next) {
  // work here
  next();
}

responseClient ko response bhejne ka object.

Project Step 5: controllers/studentController.js

controllers/studentController.js 
const students = require("../data/students");
const validateStudent = require("../validators/studentValidator");

function getAllStudents(request, response) {
  response.status(200).json({
    success: true,
    count: students.length,
    data: students
  });
}

function getStudentById(request, response) {
  const studentId = Number(request.params.id);
  const student = students.find(function (item) {
    return item.id === studentId;
  });

  if (!student) {
    return response.status(404).json({
      success: false,
      message: "Student not found"
    });
  }

  response.status(200).json({
    success: true,
    data: student
  });
}

function createStudent(request, response) {
  const validationError = validateStudent(request.body);

  if (validationError) {
    return response.status(400).json({
      success: false,
      message: validationError
    });
  }

  const newStudent = {
    id: students.length + 1,
    name: request.body.name.trim(),
    email: request.body.email.trim(),
    course: request.body.course.trim()
  };

  students.push(newStudent);

  response.status(201).json({
    success: true,
    message: "Student created successfully",
    data: newStudent
  });
}

function updateStudent(request, response) {
  const studentId = Number(request.params.id);
  const student = students.find(function (item) {
    return item.id === studentId;
  });

  if (!student) {
    return response.status(404).json({
      success: false,
      message: "Student not found"
    });
  }

  const validationError = validateStudent(request.body);

  if (validationError) {
    return response.status(400).json({
      success: false,
      message: validationError
    });
  }

  student.name = request.body.name.trim();
  student.email = request.body.email.trim();
  student.course = request.body.course.trim();

  response.status(200).json({
    success: true,
    message: "Student updated successfully",
    data: student
  });
}

function deleteStudent(request, response) {
  const studentId = Number(request.params.id);
  const studentIndex = students.findIndex(function (item) {
    return item.id === studentId;
  });

  if (studentIndex === -1) {
    return response.status(404).json({
      success: false,
      message: "Student not found"
    });
  }

  const deletedStudent = students.splice(studentIndex, 1);

  response.status(200).json({
    success: true,
    message: "Student deleted successfully",
    data: deletedStudent[0]
  });
}

module.exports = {
  getAllStudents,
  getStudentById,
  createStudent,
  updateStudent,
  deleteStudent
};

Project Step 8: server.js

server.js 
const express = require("express");
const studentRoutes = require("./routes/studentRoutes");
const logger = require("./middleware/logger");
const notFound = require("./middleware/notFound");
const errorHandler = require("./middleware/errorHandler");

const app = express();
const PORT = 3000;

app.use(express.json());
app.use(logger);

app.get("/", function (request, response) {
  response.status(200).json({
    success: true,
    message: "Welcome to Student Management API",
    endpoints: {
      students: "/api/students",
      singleStudent: "/api/students/:id"
    }
  });
});

app.use("/api/students", studentRoutes);

app.use(notFound);
app.use(errorHandler);

app.listen(PORT, function () {
  console.log("Student API running on http://localhost:" + PORT);
});
